GitHub Pages & Hexo

發表於 2020-07-31 分類於 使用 GitHub Pages & Hexo 建立個人Blog

這邊分享與紀錄了我製作這個Blog的一些過程
除了可以讓有需要的人參考，主要是留一份筆記給自己
如果有錯誤的地方也都歡迎下面留言指正~

GitHub是什麼?

Git是一個工具，讓你可以進行版本控制(就是所謂的軟體更新v1 v2 v3….)
GitHub是一個網站，讓你的專案可以放在網站上與其他工程師共享或是與你的組員共同完成專案，也可以去學習、探索其他大神們的作品

Github Page是什麼?

GitHub Page提供了免費的域名讓你上傳你完成的專案，一個人甚至可以擁有多個GitHub Page，但是前提是你放上去的專案只能呈現靜態的(HTML、CSS都算是靜態的)，動態的無法展示(有任何需要HTTP Request之類的網站)，如果想要自己的域名，GitHub Page上也可以使用第三方的域名。
!!注意!!GitHub Page中一個目錄的檔案存放上限為1GB. 流量限制為1 個月100GB，所以建議需要存放大量圖片者不要放在專案的目錄中，可以善用網路工具，如Google相簿等，後面會詳述。

Hexo是什麼?

Hexo 是一個快速、簡單且強大的網誌框架。Hexo 使用 Markdown（或其他標記語言）解析您的文章，並在幾秒鐘內，透過漂亮的主題產生靜態檔案。(來自Hexo官網的介紹)
簡單來說就是透過Hexo讓原本只是一個靜態網頁的GitHub Page可以像一個Blog一樣運行，詳細的說明可以參考 https://www.zhihu.com/question/51588481 ，裡面也有比我講得更詳細的敘述。

我需要什麼?

  1. Local本機環境需要安裝
    (1) Node.js
    (2) Git

  2. 註冊一個GitHub帳號
    https://github.com/join

  3. 一個順手的文字編輯器，我使用的是Notepad++
    https://notepad-plus-plus.org/downloads/

  4. 安裝Hexo
    打開CMD輸入指令

  5. 建立一個專案

(1) 在Github上Start a project

(2) 替你的Repostitory命名並Create

(3) 建立完成後進入到Setting，找到下面的GitHub Pages並且選擇Choose a theme

(4) 隨便選擇一個主題，按下Select theme，這個主題只是暫時的

(5) 輸入.github.io/就可以看到剛剛建立的GitHub Pages

到目前為止我們的前置作業就算是完成了~
