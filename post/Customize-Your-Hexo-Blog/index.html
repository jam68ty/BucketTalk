個人化你的Hexo Blog

發表於 2020-07-31 更新於 2020-08-17 分類於 使用 GitHub Pages & Hexo 建立個人Blog

Blog的個人化方法有太多，實在是沒有辦法用幾篇文章就說明白
更何況我自己也還沒有全部試過呢~
在網路上找到不錯的資源也都可以試著實作
當然要記得自己改了什麼東西，免得為了試新功能導致原本的專案毀損
這就牽扯到hexo的master跟branch了，這些以後有空再來研究看看吧~
這邊我們先從Hexo的主題套用說起!

Hexo預設使用的主題是landscape，我們在專案的themes資料夾中可以看到這個主題檔案
目前最多人使用的主題是: NexT
這個主題的文件十分清楚且可以進行很多個人化的設置，所以這邊以套用這個主題為例
另外這邊也有很多主題可以挑選使用：https://hexo.io/themes/

複製主題檔

首先我們進到themes資料夾下並複製NexT的檔案下來，結果可以看到目錄下多了一個主題hexo-theme-next

另外git clone完後記得回到上一層目錄

NexT主題下的個人化

NexT除了這種版型之外也有很多可以選擇的，在themes/hexo-theme-next/_config.yml中可以修改。
找到Scheme，將註解的#取消掉，只留下一個沒有註解。再次執行hexo cl & hexo g & hexo s。

可以看到主題版型改變了，可以嘗試後先執行hexo s後最後選擇自己喜歡的版型再佈署上去

Cleans the cache file (db.json) and generated files (public).

另外，通常在修改後要重新佈署上去，我都會先下 hexo cl 的指令清除快取，以確保hexo沒有太舊的設定或是檔案導致修改失敗
